1

Create Supabase Project

Go to supabase.com → New Project. Enable Email Auth under Authentication → Providers. Copy your Project URL and anon key from Project Settings → API.

2

Paste Your Credentials

Open this HTML file, find the config block at the top of the <script> section (search for SUPABASE_URL), and replace the two placeholder values with your actual Project URL and anon key.

3

Run the SQL Schema

Click Copy SQL Schema below, then paste into your Supabase SQL Editor and run it. This creates all 8 tables with RLS policies already configured.

or expand below 

    

    
4
Create Staff Accounts
In Supabase → Authentication → Users → Invite user. Then in the profiles table, insert a row with id = the user's UUID, role = 'registrar' or 'staff', and staff_id.

    
5
Import Existing Data
Go to Supabase → Table Editor → Import Data (CSV) for each table. Import order: degree_programs → courses → students → enrollments → grades → attendance. Use the CSV templates in each section of this app.

    
6
Email Notifications (optional)
Deploy a Supabase Edge Function send-notification using Resend or SendGrid. Set the secret: supabase secrets set RESEND_API_KEY=re_xxxx. The email log table already tracks all notification events.

    
7
Required SQL Migrations (run once)

      
Run these in your Supabase SQL Editor to enable photo upload approval and gender/Burmese name features:

      
-- Photo URL column for edit requests (required for photo upload approval)
ALTER TABLE student_edit_requests ADD COLUMN IF NOT EXISTS photo_url text;

-- Gender column for students (Male/Female section grouping)
ALTER TABLE students ADD COLUMN IF NOT EXISTS gender text;

-- Burmese name column for degree programs
ALTER TABLE degree_programs ADD COLUMN IF NOT EXISTS name_my text;

-- Linked program column for academic years (allows filtering years per degree)
ALTER TABLE academic_years ADD COLUMN IF NOT EXISTS program_id text REFERENCES degree_programs(id) ON DELETE SET NULL;

-- Upload-photo Netlify function: deploy netlify/functions/upload-photo.js
-- This function handles student photo submissions and creates pending edit requests.
